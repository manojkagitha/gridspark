# Blog Drafts

Drop your `.docx` blog post files here.

## How it works

1. Add your `.docx` file(s) to this folder.
2. Tell Copilot: _"Convert the drafts in `_drafts/` to blog posts"_
3. Copilot will read the content, format it as a proper JS article file, add it to the correct category under `src/lib/articles/`, and update the category `index.js`.

## What to include in your Word doc (recommended)

| Field | Example |
|---|---|
| **Title** | The Future of DevOps in 2026 |
| **Category** | `devops` / `ai` / `cloud` / `frontend` / `backend` / `data` / `ml` / `security` / `testing` / `blockchain` |
| **Author name** | Jane Smith |
| **Author role** | Senior DevOps Engineer |
| **Author bio** | One or two sentences |
| **Excerpt** | 1–2 sentence summary used on the blog listing page |
| **Read time** | e.g. 8 min read |
| **Main body** | The full article content |
| **References** _(optional)_ | List of links / sources cited |

Any field you leave out will be filled in with a sensible default.
